## Algorithm Contents

### 자료 구조(Data Structure)

<img src="https://user-images.githubusercontent.com/111719007/227140899-39a03808-be67-4be8-96e7-9d418f8f8ba9.png" height="250"/>

- `자료 구조`는 컴퓨터 과학에서 `효율`적인 `접근` 및 `수정`을 가능케 하는 `자료`의 `조직`, `관리`, `저장`을 의미함
- 다시 말하면, `데이터`를 얼마나 `효율적`으로 `저장 관리`하고 `메모리`를 `절약`하거나 `실행시간`을 `단축`시키는 목적임
- 자료 구조의 종류로는 위의 표를 보면 알수 있듯이, 크게 `선형구조`와 `비선형구조`로 나뉘어 있음
- `선형구조`는 데이터 요소들이 `일렬`로 연결되어 있는 `구조` (데이터 요소들 사이의 `관계`가 `1대1`)
- `비선형구조`는 데이터 요소들이 `계층적`으로 구성되어 있는 `구조` (데이터 요소들 사이의 `관계`가 `1대N` 또는 `N대N`)

#### 종류 - 선형구조
1. 리스트: 데이터 요소들이 일렬로 연결되어 있는 구조
2. 스택: 데이터를 스택의 맨 위에 삽입(push), 맨 위에 있는 데이터를 삭제(pop), 삽입과 삭제가 스택의 한쪽 끝에서만 이루어짐.
3. 큐: 데이터를 큐의 뒤쪽에 삽입(enqueue), 앞쪽에서 데이터를 삭제(dequeue), 삽입은 뒤쪽에서, 삭제는 앞쪽에서 이루어짐.
4. 덱: 양쪽 끝에서 삽입과 삭제가 가능한 구조, 맨 앞쪽에 데이터를 삽입(push_front)하거나 삭제(pop_front)가 가능하며, 맨 뒤쪽에 데이터를 삽입(push_back)하거나 삭제(pop_back)할 수 있
5. 연결리스트: 데이터 요소들이 일렬로 연결되어 있지만, 노드가 포인터를 이용해 서로 연결된 구조

#### 종류 - 비선형구조
1. 트리: 노드들이 계층적으로 연결된 비선형 구조, 한 노드가 다수의 자식 노드를 가질 수 있지만, 부모 노드로 되돌아갈 수 없는 구조를 가지며, 최상위 노드는 루트(Root) 노드
2. 그래프: 노드와 노드를 연결하는 간선(Edge)으로 이루어진 비선형 구조, 간선은 방향 그래프와 무방향 그래프로 나뉘며, 노드는 하나 이상의 간선으로 연결될 수 있음
