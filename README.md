### 자료 구조(Data Structure)란 무엇인가??

- `자료 구조`는 컴퓨터 과학에서 `효율`적인 `접근` 및 `수정`을 가능케 하는 `자료`의 `조직`, `관리`, `저장`을 의미함
- 다시 말하면, `데이터`를 얼마나 `효율적`으로 `저장 관리`하고 `메모리`를 `절약`하거나 `실행시간`을 `단축`시키는 목적임
- 자료 구조의 종류로는 위의 표를 보면 알수 있듯이, 크게 `선형구조`와 `비선형구조`로 나뉘어 있음
- `선형구조`는 데이터 요소들이 `일렬`로 연결되어 있는 `구조` (데이터 요소들 사이의 `관계`가 `1대1`)
- `비선형구조`는 데이터 요소들이 `계층적`으로 구성되어 있는 `구조` (데이터 요소들 사이의 `관계`가 `1대N` 또는 `N대N`)

#### 선형구조
- 리스트: 데이터 요소들이 일렬로 연결되어 있는 구조
- 스택: 데이터를 스택의 맨 위에 삽입(push), 맨 위에 있는 데이터를 삭제(pop), 삽입과 삭제가 스택의 한쪽 끝에서만 이루어짐.
- 큐: 데이터를 큐의 뒤쪽에 삽입(enqueue), 앞쪽에서 데이터를 삭제(dequeue), 삽입은 뒤쪽에서, 삭제는 앞쪽에서 이루어짐.
- 덱: 양쪽 끝에서 삽입과 삭제가 가능한 구조, 맨 앞쪽에 데이터를 삽입(push_front)하거나 삭제(pop_front)가 가능하며, 맨 뒤쪽에 데이터를 삽입(push_back)하거나 삭제(pop_back)할 수 있
- 연결리스트: 데이터 요소들이 일렬로 연결되어 있지만, 노드가 포인터를 이용해 서로 연결된 구조

#### 비선형구조
- 트리: 노드들이 계층적으로 연결된 비선형 구조, 한 노드가 다수의 자식 노드를 가질 수 있지만, 부모 노드로 되돌아갈 수 없는 구조를 가지며, 최상위 노드는 루트(Root) 노드
- 그래프: 노드와 노드를 연결하는 간선(Edge)으로 이루어진 비선형 구조, 간선은 방향 그래프와 무방향 그래프로 나뉘며, 노드는 하나 이상의 간선으로 연결될 수 있음

#### 자료구조의 특징
- 효율성: 데이터를 효율적으로 저장, 검색, 정렬, 삭제하는 데 있어서 시간 및 공간 복잡도를 고려해야 하며, 적합한 자료구조를 선택하고 적용함으로써 데이터 처리의 효율성을 높임.
- 추상화: 복잡한 데이터를 간단한 모델로 추상화하여 표현하여, 프로그래밍을 보다 편리하게함.
- 재사용성: 한번 구현된 자료구조는 다양한 알고리즘에서 사용될 수 있으며, 라이브러리 형태로 제공될 경우 개발 시 편리함.


## Algorithm Contents

### 알고리즘(Algorithm)이란 ?? 
- 알고리즘(Algorithm)은 입력값을 받아서, 원하는 결과를 출력하기 위한 일련의 단계적인 절차를 의미
- 즉, 어떤 문제를 해결하기 위한 명확하고 구체적인 해결책이며, 컴퓨터 프로그래밍, 데이터 분석, 인공지능 등 다양한 분야에서 활용
- 일반적으로 순서도, 유사코드, 자연어, 프로그래밍 언어 등 다양한 방법으로 표현
- 알고리즘을 코드로 작성하기 위해서는 먼저 해당 알고리즘을 분석하고 이해한 후에, 프로그래밍 언어로 구현합니다
- 입력값 -> 변수와 데이터구조 초기화 -> 알고리즘의 단계를 코드로 표현 -> 실행 -> 출력
